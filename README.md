# Class Homework Builder

> Personalized homework from a textbook PDF. Pick a chapter section, get a multi-phase study packet — preview, flashcards, mini-games, a boss-fight quiz, real-life examples, reflection — generated by Gemini and rendered as an interactive SPA.

[![Build and push Docker image](https://github.com/ganiyevuz/class-homework-builder/actions/workflows/docker-publish.yml/badge.svg)](https://github.com/ganiyevuz/class-homework-builder/actions/workflows/docker-publish.yml)

---

## What it does

1. Upload a curriculum textbook PDF (biology, English, history, math, geometry, chemistry, physics).
2. Gemini extracts the table of contents; pick any section.
3. The pipeline runs 8–10 phases in parallel — distilling the lesson, classifying difficulty (EASY / HARD), then producing structured content for each phase.
4. The student opens the result and works through it: flippable flashcard deck, rapid-fire memory sprint, four game types (adaptive quiz / tile match / memory match / sentence fill), a HP-based boss fight, comprehension reading, and a reflection close.
5. Everything downloads as a ZIP (`homework.md` + 5 structured JSON files).

Subjects ship with their own prompt templates in Uzbek (with English on the side for ESL).

## Why it's interesting

- **Schema-constrained generation, not parsing.** Every interactive phase (flashcards, memory-sprint, games, boss fight, reading) returns Pydantic-validated JSON via Gemini's `response_schema`. No regex post-processing.
- **DAG-parallel pipeline.** Phases run concurrently when their dependencies are satisfied. Same-image-no-config-change wall-clock dropped from ~156s to ~60–90s on HARD jobs.
- **Postgres-backed job queue** using `SELECT ... FOR UPDATE SKIP LOCKED`. Workers run embedded in the API process or standalone via `python -m app.services.worker`. Restart-safe (no orphaned jobs), retries with exponential backoff, queue-depth backpressure.
- **Per-job Gemini context cache.** Lesson context + universal SVG rules + no-preamble directive packed into a Gemini context cache; every content phase references it at ~25% input billing rate. Combined with cross-job extract caching, ~70% of input tokens hit cache.
- **Three-layer idempotency on `/generate`.** `Idempotency-Key` header + natural-key dedup + Postgres advisory lock. Double-clicks, network retries, and concurrent races all return the same job.
- **Token auth** via `Authorization: Bearer` header (REST) + `?token=` query param (SSE — `EventSource` can't set headers). SPA stores the token in `sessionStorage`, route guard bounces unauth'd traffic to `/login`.

## Architecture at a glance

```
                       ┌──────────────────┐
        student SPA    │  FastAPI process │   ← optionally embeds queue worker
       ─────────────►  │  /api/v1/...     │
                       │  /preview/:id    │
                       └────────┬─────────┘
                                │ INSERT homework_jobs
                                │   status='pending'
                                ▼
                       ┌────────────────────────────────────┐
                       │       Postgres                     │
                       │   homework_jobs (the queue)        │
                       │   FOR UPDATE SKIP LOCKED dispatch  │
                       └────────────┬───────────────────────┘
                                    │
                  claim + run pipeline.run(job_id)
                                    │
            ┌───────────────────────┼───────────────────────┐
            ▼                       ▼                       ▼
   ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
   │  Worker pod 1   │   │  Worker pod 2   │   │  Worker pod N   │
   │  concurrency=4  │   │  concurrency=4  │   │  concurrency=4  │
   └────────┬────────┘   └────────┬────────┘   └────────┬────────┘
            │                     │                     │
            └─────────────────────┴─────────────────────┘
                                  │
                                  ▼  Gemini API (semaphore-capped)
                  extract → classify → [parallel content phases] → assemble
```

The pipeline lives entirely server-side; the SPA streams progress via SSE and renders interactive components (boss fight, flashcards, etc.) directly from the JSON columns.

## Quick start (local, 30 seconds)

```bash
git clone https://github.com/ganiyevuz/class-homework-builder.git
cd class-homework-builder
cp .env.example .env
# edit .env: set GEMINI_API_KEY=...
docker compose up --build
```

Open `http://localhost:8000`. Compose runs Postgres + migrations + API + embedded worker in one command. Persistent Postgres volume (`edu_pgdata`) — `docker compose down -v` to wipe.

For development without Docker: see [DEPLOY.md § Required environment variables](./DEPLOY.md) for env setup, then `uv sync` + `alembic upgrade head` + `uvicorn main:app --reload`.

## Tech stack

| Layer | Stack |
|---|---|
| **Backend** | FastAPI · async SQLAlchemy 2.0 · asyncpg · Pydantic v2 · loguru · alembic |
| **LLM** | Gemini (google-genai SDK, async) · `response_schema`-constrained JSON · context caching |
| **Frontend** | React 19 · Vite 8 · TypeScript · Tailwind v4 (OKLCH) · TanStack Query · `react-markdown + rehype-raw` |
| **DB** | Postgres 16 (job queue, structured payloads in JSONB, advisory locks) |
| **Build & ship** | uv · Docker (multi-stage, multi-arch) · GitHub Actions (build + push) |

## Configuration

All settings via env vars; defaults in [`app/config.py`](./app/config.py). See [`.env.example`](./.env.example) for the full annotated list. Production essentials:

| | |
|---|---|
| `GEMINI_API_KEY` | Required. https://aistudio.google.com/apikey |
| `GEMINI_MODEL` | Pin to a stable release (avoid `*-preview` / `*-exp`) |
| `AUTH_TOKEN` | Strongly recommended in prod; comma-sep for multi-tenant |
| `WORKER_CONCURRENCY` | Embedded worker concurrency. Set `0` in API-only pods |
| `GEMINI_MAX_CONCURRENCY` | Process-wide cap on Gemini calls; tune to your tier |
| `JOB_TIMEOUT_SECONDS` | Hard ceiling per pipeline run |
| `QUEUE_BACKPRESSURE_LIMIT` | `503` when queue depth exceeds; `0` disables |

## Deployment

Two topologies, same Docker image:

- **Single pod** (≤1K jobs/day): API + embedded worker. `uvicorn main:app`.
- **Horizontal** (more throughput): API pods (`WORKER_CONCURRENCY=0`) + worker pods (`python -m app.services.worker`). Same image, different command.

CI publishes to Docker Hub on push to `master` / `v*.*.*` tags. Multi-arch (amd64 + arm64), GHA-cache, SBOM + provenance.

Full guide with cloud recipes (Render, Fly.io, Cloud Run, ECS), pre-launch checklist, and operational gotchas: **[DEPLOY.md](./DEPLOY.md)**.
