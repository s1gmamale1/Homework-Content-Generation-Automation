01 / executive decision

Keep the architecture. Fix four things.

The two-package split, additive migration, provider gateway, phase-DAG, structured validation, and the banned-legacy-name QA fence all survive. Four corrections stop the plan from silently dropping content or emitting JSON the platform rejects.

#CorrectionWhy it's load-bearing
1Don't flip the generator to runtime-v2 default until the platform can render Case-Based Preview + Unlock Gate.The runtime has no CBP surface yet — defaulting now ships an invisible Preview.
2Canonical schemas may be richer; beta export must obey the platform's constrained schemas exactly.A constrained Literal[...] field rejects extra values even under extra="allow".
3Case-Based Preview must carry a separate decision_process_explanation slot.Required by CBP standard v1.1 (slot 7) — the commit-before-consequence reasoning step.
4Platform repo + project standards are the prompt / source-contract authority.A forked prompts_v2/ island guarantees drift from the live standard.
Verdict — the plan is strong; it just needed a seatbelt before the engine got strapped to a rocket. Build canonical now, export safe now, block the runtime-v2 default until the platform catches up.
02 / what the corrections prevent

Two failure modes, now blocked.

Both were silent in the original plan. Each is now closed by a flag plus a test in PR-7.

FAILURE MODE 1
Silent content loss
generator emits CBP → runtime ignores it
  • No case_based_preview field in platform ContentJSON
  • No CBP injector constant — nothing renders it
  • extra="allow" stores the key, then it vanishes from the student view
Fix: CBP withheld from beta export until PLATFORM_CBP_RUNTIME_READY=true
FAILURE MODE 2
Hard validation failure
generator emits richer shapes → platform rejects
  • MemoryCheck type is Literal[mcq, fill_blank, choose_explanation]
  • RLC validator demands exactly 5 ordered steps
  • 8-type MemoryCheck or loose RLC → ValidationError on ingest
Fix: down-map + bind-to-contract, asserted by a real-schema conformance test

06 / the load-bearing content rule

Case-Based Preview lives or dies on slot 7.

Three recognition checkpoints carry low friction. The production reasoning sits in the Decision Process Explanation — open-ended, placed after checkpoint 3 and before the consequence, so the student commits before the outcome can be reverse-rationalized.

Required CBP structure · 10 slots
DPE = non-negotiable
01case_setup
02 · recognizecheckpoint_1 · identify
03learning_block_1
04 · recognizecheckpoint_2 · decide
05learning_block_2
06 · recognizecheckpoint_3 · justify
07 · PRODUCEdecision_process_explanation
08final_simulation
09feedback_summary
10completion_rules
QA hard-fails → DPE missing DPE has options DPE passable in one word DPE after consequence
Reverse-test caveat — the Practice Arc reverse-test (same story, new numbers, infer the unnamed formula) only adapts into the platform's 5-step RLC as a compatibility bridge. That shape's concept_select hands students concept chips to recognize from — it does not yet express "infer without labels." Don't call it a faithful runtime port until the platform adds an infer_method / infer_formula step kind.
