## Identity

NETS Homework is a **Buzan-Based Guided Learning Journey** built on official textbook content.

NETS does not replace the textbook. The textbook remains the source and reference. NETS turns textbook material into a better homework journey.

Short form: **case-based learning → memory anchors → recall readiness → conceptual practice → boss mastery → reflection and feedback.**

---

## Flow

```
Homework Tab
│
├── Learning Sections
│   ├── Case-Based Preview / Real-Life Learning Case
│   └── Flashcard Learning
│       └── Quizlet-Style Test / Memory Check
│
├── Unlock Gate
│   └── Practice Arc unlocks when learning sections meet minimum requirements
│
├── Practice Arc
│   ├── Gamified Conceptual Practices
│   ├── Interactive Games
│   ├── Simulations
│   └── Boss Arena
│
└── Reflection / Debrief / Marking
    ├── Student reflection questions
    ├── AI/system feedback
    ├── Score and weak points
    ├── Passed / Not Passed state
    └── Redo route if needed
```

## Learning Sections

The two Learning Sections (Case-Based Preview and Flashcard Learning) are independently accessible and completable in **any order**. They teach different aspects of the lesson and have no dependency on each other. Both must satisfy minimum readiness requirements before the Unlock Gate opens.

---

### Case-Based Preview / Real-Life Learning Case

**Role:** The NETS learning layer built from textbook material. Teaches the concept through a real-life case, not a passive reading blob.

**Includes:**
- Textbook summary
- Story / case setup, real-life framing
- Decision checkpoints
- SVG / animation / simulation moments where useful
- AI feedback on decision-making
- Summary of thought process

**Student does:** Learns through story/case, makes decisions, sees consequences.

**Feel:** *I just learned this idea, now I immediately use it in a realistic situation.*

**Difficulty:** Guided and low-to-medium stakes. Tests fresh understanding and meaning-building, not final mastery.

**Forbidden:**
- Forbid Case Study that loses the original textbook concept.
- Forbid Case Study without checkpoint-based learning.
- Forbid Case Study where the student is not the decision-maker / solver.
- Forbid checkpoint decisions with no consequence.

---

### Flashcard Learning

**Role:** Mechanical memorization of essential terms, formulas, vocabulary, definitions, symbols, and anchors.

**Student does:** Reviews flashcards freely. No time limit. Scrolls and revisits as needed. Memorizes key cards.

**Constraint:** Must view all flashcards at least once before the required Memory Check.

**Forbidden:**
- Forbid Flashcards being counted as conceptual mastery.
- Forbid Flashcards completing the learning branch without Memory Check.

---

### Quizlet-Style Test / Memory Check

**Role:** Checks memorization readiness from flashcards. Required after flashcard review.

**Question types:** True/False, Multiple Choice, Tile Match, Term ↔ Definition, Formula ↔ Name, Vocabulary ↔ Meaning.

**Answers:** *Do you know the key cards?* — not *Can you apply the concept deeply?*

**Forbidden:**
- Forbid Memory Check replacing conceptual Practice Arc.

---

### Learning Section Ordering

**Forbidden:**
- Forbid forced order between Case Study and Flashcards.

---

## Unlock Gate

**Role:** Separates learning preparation from actual homework practice. Prevents blind practice.

**Unlocks when:**
- Case-Based Preview completed with minimum acceptable result
- Flashcard Learning viewed at least once
- Memory Check completed with minimum acceptable result

**UI:** Chain-breaking animation. Button wording: *Enter Practice Arc* / *Start Practice Arena* / *Practice Arc Unlocked*. Never *Start Homework* — student is already inside the Homework Tab.

**Forbidden:**
- Forbid unlocking Practice Arc without both learning branches completed.

---

## Practice Arc

**Role:** The actual homework body. Where games, interactivity, and simulations live.

**Tests:** Conceptual understanding, critical thinking, debugging, error detection, calculation, issue → formula conversion, proof / reasoning, simulation-based application, subject-specific applied skills.

**Feel:** One full mission arc leading to one Boss Arena.

**Minimum conceptual structure:**
- Full / hard homework: Practice Mission(s) → Simulation / Medium-Hard Challenge → Boss Arena
- Lighter / easy homework: At least one strong conceptual practice + mini-boss or final challenge

**Forbidden:**
- Forbid Practice Arc as random disconnected games.
- Forbid Practice Arc focusing mainly on memorization.
- Forbid Practice tasks that do not match target skill.

---

## Boss Arena

**Role:** Mastery peak of the Practice Arc.

**Locked reasoning skeleton:** Why → How → What.
- Why is this formula / method needed?
- How do you apply it?
- What does the result mean?

**Tests:** Conceptual understanding, application, transfer, explanation, reasoning chain, weak points from previous activities, integrated lesson mastery.

**Forbidden:**
- Forbid Boss as quiz + HP skin.
- Forbid Boss without Why → How → What reasoning.
- Forbid Boss testing only flashcard recall.

---

## Reflection / Debrief / Marking

**Role:** Short debrief plus feedback / marking input. Closure and improvement route. Not a long essay, not a generic closing ritual.

**Student may answer:**
- What was hardest?
- Why did you make your main decision?
- What mistake would you avoid next time?
- What concept finally clicked?

**System / AI outputs:** Score, feedback, weak points, strong points, redo recommendation, passed / not passed status, next-step suggestion.

**Pass / retake terminology:** Use *Attempt completed, homework not passed* or *Needs Retry*. Avoid *Not Completed* alone — the student did attempt the homework.

**Retake rule:** Same concepts, not exact same questions. Prevents memorization farming.

**Forbidden:**
- Forbid generic Reflection disconnected from actual performance.
- Forbid exact-question retake farming.
- Forbid *Not Completed* as the only label after a real attempt.

---

## Buzan Distribution

Buzan techniques are not isolated in a single phase. They run as a cross-flow foundation.

| Section | Buzan Role |
|---|---|
| Case-Based Preview | Story association, visual hooks, simplified concept maps |
| Flashcard Learning | Memory anchors, key terms, formulas, vocabulary |
| Memory Check | Active recall, retrieval, pattern reinforcement |
| Practice Arc | Pattern recognition, mistake linking, applied association |
| Boss Arena | Why → How → What chain as mastery reasoning pattern |
| Reflection | Weak-point memory recap, next anchor, learning loop closure |

The old standalone Consolidation phase is **superseded**. Anchor Lock / Memory Lock behavior is redistributed across Flashcards + Memory Check + Reflection, and optionally inside Practice Arc when the lesson requires it.
