# Project Memory — Master Log

Full record of every task done on this project. Quick-scan titles live in [INDEX.md](./INDEX.md).

## How this works
- Every completed task gets **one entry here** (full detail) and **one line in INDEX.md** (title only).
- IDs are sequential 4-digit (`0001`, `0002`, …). The INDEX row and the master heading share the same ID.
- Newest entries go at the **bottom**.
- **Rollover rule:** when this file passes ~500 lines, the oldest entries move into `docs/memory/archive/MASTER_MEMORY_<range>.md` and INDEX rows are updated to point there. This honors the CLAUDE.md "files under 500 lines" rule and keeps the active log small enough to read into context.

## Entry template
```
## [NNNN] Title — YYYY-MM-DD (branch)
**What:** what was actually done
**Why:** the reason / goal
**Files:** paths touched
**Verified:** how it was confirmed (command run + result), or "not verified — <reason>"
**Notes:** decisions, tradeoffs, follow-ups, links to [[related]]
```
