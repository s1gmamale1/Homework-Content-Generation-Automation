04 / not infrastructure

Three content rules that stay.

These ride inside the prompts/schemas — not a separate QA stage (QA dropped earlier as technical).

01
Decision Process Explanation (CBP slot 7). Required by CBP v1.1. Open-ended, before the consequence, evaluates concept · method · mistake. Never an MCQ. Method stays unnamed so the student commits first.
02
Source fidelity. Every phase/game traces to source-map concept IDs. No invented facts.
03
Spec conformance + fail-fast. Each phase/game matches its authored prompt spec; a phase in a sequence with no prompt file errors rather than improvising.
Open question — if Infra.zip holds authored Flow v2 prompt specs, they get ported into prompts/<subject>/ and registered in flows.py. Share it and the prompt coverage can be verified phase-by-phase against the supported subjects.
