00 / the correction

Wrong scope → right scope.

The whole framing shifted from "generator that must stay compatible with the Homeworks runtime" to "content factory that owns nothing downstream."

01 / what the generator does

One chain, content only.

Source fidelity is the root; after the source map the generators fan out in parallel. The team can start isolated generators today against a mock source map.

book / textbook
  → locate chapter + section
  → extract + normalize section content
  → build source map          # factual anchor · concept IDs
  → map concepts to Flow v2 divisions
  → run correct prompt per phase/game   # from Infra registry, never invented
  → generate content                    # CBP · flashcards · 6 games · boss · memory check
  → reflection / marking + weak-point signals
  → assemble homework-content.md
  → content QA vs Flow v2 specs
02 / outputs per job

Four artifacts. No export gymnastics.

There is no beta-safe layer and no future-runtime layer — those collapsed. The Markdown is the human handoff; the JSON is the structured record.

Every generation job produces exactly these
  • content_package.jsoncanonical Flow v2 content
  • homework-content.mdhuman-readable handoff, mirrors the package, content only
  • flow_manifest.jsonprompt paths / versions / hashes, model usage, generation order
  • qa_report.jsoncontent QA results
Content convention — answer keys, rubrics and teacher notes are labeled distinctly from student-visible text so any downstream consumer can split them. This replaces the deleted browser-stripping rule — it's content hygiene, not a security control.
03 / not deleted by accident

Three rules that survived the cut.

These read like platform caveats but are actually content standards. They stay — reframed from platform-schema to content-spec.

01
Decision Process Explanation (CBP slot 7). Required by CBP standard v1.1. Open-ended, before the consequence, evaluating concept · method · mistake. Never an MCQ. The method stays unnamed in the case body so the student commits first.
02
Source fidelity. Every generated phase/game traces back to extracted source concept IDs. No invented facts. Extraction returning unrelated content stops the job.
03
Each phase/game conforms to its Infra spec. This is the content contract — not a platform Pydantic validator. So Memory Check item types and RLC structure follow the content spec; the reverse-test is authored straight from the RLC spec with no "5-step bridge" caveat.
