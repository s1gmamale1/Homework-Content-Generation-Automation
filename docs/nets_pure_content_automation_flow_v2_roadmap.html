00 / scope correction

What this is — and what got cut.

The earlier docs cooked in runtime/platform scope. None of it blocks this team. Removed for good; the right side is gone from every phase.

  • Book / textbook input
  • Chapter + section extraction
  • Source map (factual anchor)

OUT OF SCOPE · DELETED
  • Homeworks runtime / student screen
  • Platform PR-0 prerequisite

05 / the content rule that survived

Case-Based Preview lives on slot 7.

A content standard (CBP v1.1), not a runtime feature. Three recognition checkpoints carry low friction; the production reasoning is the Decision Process Explanation, placed before the consequence so the student commits first.

Checkpoints 1–3: identify · decide · justify_or_avoid_mistake as mcq/choice/short_select. No 4th checkpoint, no checkpoint-as-essay. The method/formula stays unnamed in the case body.
06 / the deliverable

homework-content.md — the human handoff.

Every job emits four content artifacts; the Markdown is the readable one. It mirrors the canonical package, includes every enabled game, and contains generated content only — never implementation code.

FOUR ARTIFACTS PER JOB
  • content_package.json — canonical Flow v2 content
  • homework-content.md — human handoff
  • flow_manifest.json — prompt paths/versions, model usage
  • qa_report.json
Content convention — answer keys, rubrics and teacher notes are labeled distinctly from student-visible text so a downstream consumer can split them. Content hygiene, not a security control.
