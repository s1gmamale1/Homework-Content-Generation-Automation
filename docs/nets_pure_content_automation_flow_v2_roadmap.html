03 / how a phase gets added

Five edits. No new infra.

Every legacy phase already proves this recipe. Adding a Flow v2 phase is five small, local edits — the scheduler, retry, usage tracking and assembly pick it up for free.

1  prompts/<subject>/<phase>.md          # the prompt, authored from the spec
2  app/schemas/<phase>.py                # Pydantic output schema
3  agent.STRUCTURED_PHASE_SCHEMAS[...]   # register phase → schema (JSON mode)
4  flows.SUBJECT_FLOWS[...]              # insert into easy / hard sequence
5  flows.PHASE_DEPS[...]                 # declare which prior outputs it consumes

05 / the content rule that stays
